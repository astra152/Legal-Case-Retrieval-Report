# case-search-report

基于用户上传的案件文件进行结构化分析、裁判规则提炼与专业报告撰写的技能框架。

## 适用场景

- 律师、法务基于案件文件（Word/PDF/TXT）生成类案检索报告
- 从多个案例中提炼裁判规则与裁判逻辑
- 生成结构清晰、论证充分的专业分析报告

## 核心能力

### 七段式报告结构

1. **分析背景与目的** — 问题来源、分析动机、研究价值
2. **文件概况** — 文件数量、案例类型、涉及法院
3. **核心规则提炼** — 裁判规则、法律要件、识别标准
4. **分类案例分析** — 案例卡片 + 裁判摘要
5. **要件拆解与矩阵** — 可视化工具（流程图/矩阵要件要素图）
6. **实务启示** — 操作建议、风险提示
7. **总结** — 核心结论、流程图/矩阵、汇总表

### 可视化工具智能选型

生成报告第五章前，必须先判断裁判逻辑类型，选择合适的可视化工具：

| 裁判逻辑类型 | 识别标志 | 适用工具 |
|-------------|---------|---------|
| 逐级筛除型 | "首先审查……其次审查……"递进式论述 | **流程图** |
| 综合权衡型 | "综合考量以下因素……"并列式论述 | **矩阵要件要素图** |
| 混合型 | 前半递进、后半并列 | **流程图嵌套矩阵** |

### 流程图设计规范

- **逐级筛除式判断树**：从默认法律立场出发，逐级设卡，"是"出结论，"否"继续深入
- **节点排序法则**：先重后轻、先原则后例外、先粗后细、先刑后民
- **终止节点三要素**：法律结论（主框）+ 适用边界（副框）+ 案例锚定

### 矩阵要件要素图设计规范

- 识别所有影响裁判的变量 → 穷尽组合 → 每种组合对应一种裁判结果
- 支持权重评分量化分析（如主观意图35% + 行为方式40% + 后果25%）

## 文件结构

```
case-search-report/
├── SKILL.md                              # 技能主文件（入口）
├── README.md                             # 本文件
├── references/
│   ├── case-analysis-methods.md          # 案例分析方法（四步法、对比法）
│   ├── flowchart-guide.md                # 可视化选型与流程图/矩阵生成指南
│   ├── report-structure.md              # 报告结构规范
```

## 支持的文件格式

| 格式 | 处理方式 |
|------|---------|
| .docx | python-docx 解析 |
| .pdf  | pymupdf 解析 |
| .txt  | 直接读取 |

## 使用方式

1. 上传一个或多个案件文件（目标案例 + 类案）
2. 系统自动解析文件，提取案号、法院、争议焦点、裁判观点、判决结果等结构化信息
3. 按七段式结构生成类案检索报告
4. 第五章自动判断裁判逻辑类型，选择流程图或矩阵要件要素图进行可视化

## 设计原则

- **不主动搜索外部案例库**，仅分析用户提供的真实案件文件
- **准确引用文件原文**，不添加未经证实的信息
- **选型先行**：先判断裁判逻辑类型，再选择可视化工具，不强行用流程图套所有场景
- **逻辑即视觉**：可视化结构 = 裁判逻辑结构
