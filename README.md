Case Search Report


法律案例分析报告自动化生成方法论 — 从案件文件到结构化报告，一键完成
它是什么


Case Search Report 是一套法律案例分析报告生成方法论，专为律师、法务人员设计。你只需上传案件文件（docx/pdf/txt），它便会自动解析、结构化提取信息，并根据案件特征动态选择最合适的报告框架，生成符合专业标准的案例分析报告。


核心定位：不主动搜索外部案例库，仅对你提供的真实案件文件进行深度结构化分析。
核心能力


能力	说明
智能文件解析	自动从 docx/pdf/txt 中提取案号、法院、案由、争议焦点、裁判要旨等结构化信息
动态框架适配	根据案件特征自动选择最合适的报告框架（问题导向型 / 标准认定型 / 矩阵分析型 / 简化型）
可视化图表	支持地区分布图、案由饼图、要件矩阵、裁判流程图（Mermaid）等多种可视化组件
多语言支持	支持英文及其他语言判决书的翻译与双语对照分析
裁判规则提炼	三层次提炼法，从案例中归纳可复用的裁判规则
完整性校验	自动检测信息缺失并标注，确保报告严谨性
报告框架


根据案件特征，动态选择以下框架之一：
框架 A：问题导向型


适用场景：用户有具体问题场景（如"大同强奸案中性行为认定标准"）


plaintext
背景与问题场景 → 案件概况 → 争议焦点与裁判规则 → 要件矩阵 → 流程图 → 实务启示 → 缺失提示

框架 B：标准认定型


适用场景：研究某罪/条款的认定标准（如"诈骗罪非法占有目的的司法认定"）


plaintext
背景与认定标准 → 案件概况 → 构成要件分析 → 要件组合矩阵 → 裁判流程图 → 学术争议 → 实务指引

框架 C：矩阵分析型


适用场景：要素繁杂、组合复杂的多维分析


plaintext
背景与研究问题 → 案件概况 → 要素识别与筛选 → 要素定义表 → 多维矩阵构建 → 裁判规则 → 流程图 → 实务启示

框架 D：简化型


适用场景：案例少、问题单一


plaintext
背景与问题 → 案件基础信息 → 核心争议与裁判 → 裁判规则归纳 → 实务建议

组件库


报告由以下可组合组件构成，根据案件特征按需选用：


组件	何时使用	核心价值
背景与场景	始终使用	建立分析起点
基础信息表	始终使用	信息完整性呈现
地区分布图	案例 ≥ 3 个	地域特征分析
案由饼图	涉及多种案由	结构分布可视化
人物关系图	用户有具体案件背景	厘清当事人关系
争议焦点分配	用户有具体问题场景	精准匹配案例
要件矩阵	要素 ≥ 3 个且需组合分析	发现裁判规律
裁判流程图	判断路径清晰可描述	可视化推理过程
规则提炼	始终使用	归纳核心裁判规则
实务指引	始终使用	提供操作建议
缺失提示	存在信息缺失时	提示补充完善
使用方式
1. 提供案件文件


上传一个或多个案件文件（支持 .docx / .pdf / .txt），并描述你的问题场景或研究背景。
2. 自动解析


脚本自动提取以下结构化信息：


json
{
  "case_name": "完整案例名称",
  "court": "裁判法院",
  "province": "所属省份",
  "trial_stage": "审判阶段",
  "doc_type": "文书类型",
  "judgment_date": "裁判日期",
  "case_number": "案号",
  "cause": "案由",
  "dispute_points": ["争议焦点列表"],
  "court_opinion": "法院认定",
  "judgment": "判决结果",
  "key_point": "裁判要旨"
}

3. 生成报告


根据案件特征和用户需求，自动选择框架并组合组件，生成完整的结构化分析报告。
设计原则


案号最重要 — 每个案例必须显示案号；无案号标注"无案号/请填写案例来源"
不可遗漏、不可捏造 — 缺失信息标注"暂未识别"或"未提供"，绝不编造
报告连贯，灵活适配 — 根据案例特点选择合适的报告框架，不生搬硬套
国内外区分 — 涉及国外判决时，国内外案件分开呈现，保留原文关键字段
